Aujourd'hui, je vais faire une brève introduction sur le système de stationnement sans homme. Ici j'ai opté Gopay mais c'est juste un exemple. On peut utiliser le système de paiement que nous allons concevoir.

1

Découvrez comment fonctionne le système de stationnement sans homme. Deux composants principaux sont requis, à savoir le matériel et le logiciel.

2

Pour le matériel, on aura besoin de barrières d'entrée et de sortie, de caméras, de kiosques de paiement, de serveurs et d'équipements réseau. Les barrières d'entrée et de sortie servent à réguler l'accès des véhicules. Les caméras sont utilisées pour la reconnaissance des caractéristiques des véhicules pour la sécurité pour le moment. Les kiosques de paiement sont utilisés pour le ticket avec code QR. Les serveurs sont employés pour conserver les données indispensables du système. Les équipements réseau permettent de connecter les différents éléments du système. À titre d'exemple, les caméras sont reliées aux serveurs via les équipements réseau... À suivre.

3

Pour le logiciel, on aura besoin d'un logiciel de gestion de stationnement, d'un logiciel de reconnaissance de plaque d'immatriculation, d'un logiciel de traitement des paiements, d'un système de gestion de base de données et d'une API GoPay par exemple.

4

Voici le flux de travail du système de stationnement sans homme. La caméra enregistre la plaque d'immatriculation du véhicule entrant et les caractéristiques du véhicule. Le client clique sur le bouton pour obtenir le ticket avec le code QR. La barrière d'entrée s'ouvre et le véhicule entre dans le parking. Au moment de la sortie, le client scanne le code QR pour payer avec GoPay. Le client obtiendra un reçu de paiement avec le code QR pour la sortie. Le client scanne le code QR pour sortir du parking. La barrière de sortie s'ouvre et le véhicule sort du parking.

5

Pour la gestion de base de données, on aura besoin de gérer les données des utilisateurs, les informations des véhicules et l'historique des paiements.

6

Pour la sécurité, on aura besoin de HTTPS, de contrôles d'accès et d'audits réguliers. Les audits réguliers sont nécessaires pour assurer la sécurité du système. Les audits veulent dire qu'on vérifie le système régulièrement pour trouver les failles de sécurité.
